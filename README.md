## Кейсы и задания

На хакатоне **6 реальных бизнес-кейсов** от компаний-партнёров. Каждый кейс — конкретная задача, которую бизнес хочет решить с помощью AI.

### Направления кейсов

| # | Направление | Область |
|---|-------------|---------|
| 1 | **AI-ассистент** — снижение нагрузки на поддержку | Ритейл / сервис |
| 2 | **Автоматизация заявок** — обработка документов и маршрутизация | B2B-услуги / логистика |
| 3 | **Предиктивная аналитика** — предсказание оттока клиентов | SaaS / финтех |
| 4 | **Рекомендации** — персонализация контента в реальном времени | E-commerce / медиа |
| 5 | **Автоматическая отчётность** — генерация сводок и аналитика | Производство / управление |
| 6 | **AI в HR** — ускорение найма от заявки до оффера | HR / профессиональные услуги |

> Подробные брифы с данными, контекстом и критериями от каждого заказчика лежат в папках `cases/case-1/`, `cases/case-2/` и т.д.

### Структура папки кейса

```
cases/case-N/
├── BRIEF.md              # Бриф от заказчика: задача, контекст, данные
├── data/                 # Датасеты и материалы от заказчика (если есть)
├── evaluation.md         # Специфические критерии оценки этого кейса
└── contacts.md           # Контакты ментора и представителя заказчика
```

## Формат сдачи проекта

Каждая команда сдаёт результат в виде:

### 1. GitHub-репозиторий

- Создайте репозиторий вашей команды (или форкните этот)
- В корне должен быть **README.md** с описанием решения (используйте шаблон из `templates/project-readme/`)
- Код должен быть запускаемым — инструкция по запуску в README
- `.env.example` с перечнем необходимых переменных окружения (без реальных ключей!)

### 2. Работающий прототип

- Демо, которое можно показать в реальном времени
- Streamlit / Gradio / веб-приложение / CLI — формат не ограничен
- Главное — **оно должно работать** на момент питча

### 3. Презентация (питч)

- Формат: 5–7 минут выступление + 3–5 минут ответы на вопросы
- Подробнее — в разделе [Презентация и питч](#-презентация-и-питч)

## Правила хакатона

### Можно

- Использовать любые языки программирования и фреймворки
- Использовать open-source библиотеки и инструменты
- Использовать предоставленные API (OpenAI, Anthropic и др.)
- Использовать pre-trained модели (Hugging Face и т.д.)
- Гуглить, читать документацию, использовать AI-ассистентов (GitHub Copilot, ChatGPT и т.д.)
- Общаться с менторами и представителями заказчиков

### Нельзя

- Использовать готовые решения, написанные до хакатона
- Копировать код из существующих проектов целиком
- Работать удалённо (хакатон оффлайн!)
- Пушить после дедлайна
- Коммитить API-ключи и другие секреты

### Интеллектуальные права

> Права на созданные решения обсуждаются индивидуально между командами и компаниями-заказчиками. Организаторы не претендуют на права участников.

## Презентация и питч

### Формат

- **Время:** 5–7 минут выступление + 3–5 минут Q&A
- **Аудитория:** жюри (эксперты + представители бизнеса) + все участники

### Что должно быть в питче

| Слайд | Содержание |
|-------|-----------|
| 1 | Название команды и решения |
| 2 | Проблема: что решаем и для кого |
| 3 | Решение: как это работает (архитектура) |
| 4 | Демо: живая демонстрация прототипа |
| 5 | Бизнес-ценность: конкретные цифры эффекта |
| 6 | Готовность: что нужно для пилотного внедрения |
| 7 | Команда и контакты |

> Шаблон презентации — в `templates/pitch-template/`.

### Советы

- **Показывайте работающий продукт**, а не слайды с описанием
- Считайте эффект в деньгах/времени/ресурсах — жюри это ценит
- Не тратьте время на введение в тему AI — жюри в контексте
- Репетируйте тайминг — 7 минут проходят быстро

## FAQ для участников

<details>
<summary><b>Могу ли я использовать ChatGPT / Copilot / Claude во время хакатона?</b></summary>

Да! AI-ассистенты — это инструмент, и вы можете использовать их для написания кода, генерации идей, отладки. Но итоговое решение должно быть создано на хакатоне, а не скопировано из готового проекта.
</details>

<details>
<summary><b>У меня нет команды, что делать?</b></summary>

Если вы зарегистрировались индивидуально — организаторы помогут вам попасть в команду при распределении по кейсам. Также можно найти команду в Telegram-чате хакатона.
</details>

<details>
<summary><b>Какой стек лучше использовать?</b></summary>

Любой, с которым ваша команда работает быстрее всего. Для AI-части большинство участников используют Python. Для демо — Streamlit, Gradio, или веб-приложение на любом фреймворке.
</details>

<details>
<summary><b>Нужно ли тренировать модели с нуля?</b></summary>

Нет. Хакатон про прикладное применение AI, а не про обучение моделей. Используйте API (GPT-4, Claude, GigaChat), pre-trained модели, fine-tuning — что подходит под задачу.
</details>

<details>
<summary><b>Как подключиться к Wi-Fi?</b></summary>

SSID и пароль указаны в разделе [Wi-Fi и доступ](#-wi-fi-и-доступ) и будут на экранах в зале.
</details>

<details>
<summary><b>Где поесть?</b></summary>

</details>

<details>
<summary><b>Что если API-ключ закончился / не работает?</b></summary>
